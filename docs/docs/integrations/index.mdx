Here you will find a quick view of available features and all our supported integrations and models for every LLM provider deployed on `langchain4j` so far.

We are making a great effort to have most of the functions enabled according to the progress and updates of the LLM providers.

## Features
- Native image: It is compatible for builds using GraalVM CE or GraalVM Oracle for [native image](https://www.graalvm.org/latest/reference-manual/native-image/) generation.
- Completion: Supports the implementation of `natural language` and `chat` models in a synchronous manner. View examples [here](/docs/tutorials/connect-to-llm)
- Streaming: Supports `streaming` the model response back for `text-completion` or `chat-completion` models handling each event in `StreamingResponseHandler<AiMessage>` class. View examples [here](/docs/tutorials/response-streaming)
- Async Completion: Provide an asynchronous version of the completion feature.
- Async Streaming: Provide an asynchronous version of the streaming feature.
- Embeddings: Supports the implementation of `text-embedding` models. Embeddings make it easy to add custom data without fine-tuning. Generally used with Retrieval-Augmented Generation (`RAG`) tools, `Vector Stores` and `Search`. View examples [here](/docs/tutorials/embedding-store)
- Image Generation: Supports the implementation of `text-to-image` models to create realistic and coherent images from scratch. View examples [here](/docs/tutorials/image-models)
- ReRanking: Understands the implementation of re-ranking models to improve created models by re-organizing their results based on certain parameters. View examples [here](/docs/tutorials/reranking-models)

of course some LLM providers offer large multimodal model (accepting text or image inputs) and it covers more than one feature.
